This is an individual assignment.

In prior assignments, you have largely been tasked with picking up the fundamentals of web and mobile development on your own, which has helped you build up a strong foundation in their principles. This assignment provides an opportunity to explore how Large Language Models (LLMs) can assist with typical interface development tasks. You will use an LLM to help you understand and apply a professional design system to a starter website. The goal is to learn how to effectively collaborate with LLMs in the design and development process. You will write less code in this assignment than past assignments, and the coding task is not as extensive. Instead, this assignment aims to provide a lens into the benefits and limitations to leveraging LLMs for developing user interfaces.

You are required to complete this assignment using ZotGPT chat. You may choose any model within ZotGPT (e.g., Claude, Gemini, or GPT-5). You're welcome to use multiple models and compare their output, but to complete the assignment you only need to use one. To summarize:

  • You may NOT use AI coding tools such as Cursor, GitHub Copilot, Claude Code, or any other AI-powered IDEs or code completion tools. These tools may be useful fur similar tasks in the future, but requiring use of a chat-based LLM will require more reflection on what is being generated.
  • You must use only the ZotGPT chat interface for all LLM interactions. Compared with public models like ChatGPT, ZotGPT has the added benefit of ensuring that your prompts remain private and are not used for future training.
  • You may NOT use LLMs to write your reflection answers. The reflection questions must be written by you, based on your own experience and thoughts.

Read the reflection questions before you start! Half of the points for this assignment relate to your reflections on the use of LLMs for interface development. Before you begin working on the assignment, you may find it helpful to read the reflection questions in the Reflection Questions Google Doc (make a copy for your own responses). These questions will guide your thinking as you work with the LLM.

Working with the LLM

You will write very little code yourself in this assignment. Instead, you will rely on ZotGPT to generate the code which applies the design system to the starter website. You may have to correct the generated code, or further refine your prompts to clarify your intentions. But compared to prior assignments, you will do significantly less coding yourself.

Throughout this assignment, use ZotGPT to help you:

  • Explain design system concepts like color systems, typography, spacing, and elevation.
  • Suggest appropriate components for your webpage (e.g., buttons, cards, navigation).
  • Generate HTML and CSS code that applies design principles.
  • Describe problems and ask for solutions.
  • Iterate on designs and refine the implementation.

A Tour of the Starter Website

A starter repository is on GitHub Classroom.

You will work with an LLM to transform this basic starter webpage by applying professional design systems and customizing it with your own creative touches. First, let's understand what you're working with.

Order Page (index.html)

The main page displays four types of widgets that customers can order: a Toy Boat, a Paper Plane, a Stuffed Animal, and a Train Car. Each widget includes:

  • An image of the widget
  • Price information
  • Rating (out of 5 stars)
  • Input fields for quantity (0-10) and custom color selection

Shipping Page (shipping.html)

After selecting widgets, customers enter their shipping information including:

  • Street address
  • City, State, and ZIP code
  • Shipping speed selection (Express, Standard, or Slow, each with different costs)

A loading indicator appears while the order is being processed (approximately 2 seconds).

Summary Page (summary.html)

Finally, the order summary displays:

  • Total number of widgets ordered
  • Breakdown by widget type (quantity and color)
  • Subtotals for each widget type
  • Shipping information and cost
  • Total cost
  • Expected arrival date

Your goal is to apply design system principles to improve the visual design and user experience of these pages while maintaining all functionality.

A note on Functionality: If you inadvertently break some functionality during your implementation, we will not deduct points. The focus of this assignment is on learning how LLMs can help you understand and apply design systems, not on debugging. Focus your energy on exploring how to effectively collaborate with LLMs in the design and development process. That said, issues that arise could be important topics to incorporate into your reflection.

Requirements

Part 1: Understanding and Applying a Design System with LLMs (2 points)

For the first part of this assignment, you will use ZotGPT to generate code which applies the Material Design design system to your starter website using an LLM to help you understand and implement the system's principles. Material Design is Google's comprehensive design system that provides guidance on components, color, typography, spacing, and interaction patterns. You can start by exploring the MDUI component library, which provides an implementation of Material Design via Web Components.

With the help of ZotGPT, implement at least 5 different elements from MDUI for the website. We recommend considering components such as layout, top app bar, buttons, cards, text fields, and select menu. You can refer to the MDUI Documentation for the complete list of components, and apply ones you think are appropriate. The Material Design components will provide a basic structure and look. In Part 2, you will customize the colors, fonts, and spacing to make the design your own.

Part 2: Customizing with Your Own Style Guide (1 point)

In this part, you will use ZotGPT to help define and apply your own custom style guide for colors, typography, and spacing. This is where you add your creative touch to the project and make the Material Design components look unique. You will primarily edit the style.css file to implement your custom design system. This part builds directly on Part 1. You should keep the Material Design components (buttons, cards, text fields, etc.) from Part 1, but customize their appearance with your own colors, fonts, and spacing. Through this, you are "reskinning" or "theming" the Material Design components with your personal aesthetic.

A reminder to focus on learning. The goal is to practice defining and applying your own design decisions with the help of an LLM. If some aspects don't work perfectly, that's okay. We're evaluating your effort and design thinking, not perfect execution. Continue documenting all your prompts to the LLM as you work on this part.

Your code generated by ZotGPT must implement the following:

  • A Color System: Create a consistent color scheme using design tokens (CSS custom properties). You should:
    • Define at least 5 colors with clear purposes, such as primary, secondary, background, text, accent in your stylesheets.
    • Use color tokens and css variables in your stylesheets (e.g., don't hardcode colors like #ff0000 in multiple places) Apply these variables throughout your CSS and use them across the components and pages.
    • Ensure text has sufficient contrast: ≥4.5:1 for normal text, ≥3:1 for large text (18px+ or 14px+ bold).

    See Material 3 Color System for design principles and Material Accessibility Guidelines for contrast requirements

  • Typography: Apply a typography system for visual clarity. You should:
    • Choose and import custom fonts. Google Fonts is one of the examples of how fonts may look like.
    • Apply different font families, sizes, and weights for different element types in your stylesheets. Create a clear visual hierarchy between at least 5 different elements such as titles, headings, body text.
    • Apply typography consistently across all pages.

    Example of typography hierarchy:

    See Material Typography Tokens for guidance.
